01 / Overview

Project at a glance

A 24-day sprint from initial commit to a feature-complete v3.7 release, published to npm.

PetForge gives terminal users a deterministic ASCII pet that levels up from real coding activity. It listens to Claude Code's five official hooks, translates each event into XP, evolves the pet across six phases, and unlocks medal-tiered achievements - hook-driven plus optional OpenTelemetry-gated tiers. A built-in HTTP+SSE server streams the same state to a phone-installable PWA. Zero telemetry, fully local.

Architecture in one sentence. A single TypeScript ESM bundle that owns one JSON state file (~/.petforge/state.json), exposes 14 CLI subcommands, runs two optional local servers (HTTP+SSE on 7878, OTel on 7879), and patches the user's Claude Code settings.json to register five event hooks. Cross-process state writes are guarded by proper-lockfile with a Windows-specific EPERM/EBUSY retry. All side effects are confined to ~/.petforge/ and ~/.claude/settings.json (the latter with a .bak trail).

02 / Release History

From V1 to V3.7 in 24 days

Genesis commit on 30 Apr 2026, npm release on 11 May 2026, V3.7 quota tracker shipped 20 May 2026.

Ship V3.7.5 - State write boundary hardening 24 May 2026

"Wipe killer" guard added at the writeStateAtomic boundary - refuses to commit a state mutation that would zero out the pet, level, or counters without an explicit sentinel. Defense against a class of subagent-induced state wipes observed in earlier sessions.

Fix V3.7.4 - Dedup by message.id + daily auto-backup 23 May 2026

JSONL ingest now deduplicates by message.id rather than line offset. ~/.petforge/state.json gets a per-day rolling backup. Web view surfaces real API cost on the activity line.

New V3.7 - petforge history + quota tracker 20 May 2026

Two big shipments. (a) Opt-in 5h / 7d rate-limit gauge with daemon co-orchestration via up --quota, mood reactivity above 80% utilization, and 6 new quota_* achievements (46 -> 52). (b) petforge history derives lifetime API spend by ingesting ~/.claude/projects/**/*.jsonl directly, with a --sync-otel flag to overwrite state.counters.otel with the JSONL ground truth.

New V3.6 - Cross-platform service install 11 May 2026 · npm publish

User-mode auto-start across Windows (schtasks), macOS (launchd LaunchAgent), and Linux (systemd --user). No admin / sudo. Per-platform ServiceManager behind an exec indirection so the orchestrator is unit-testable without invoking real binaries. First npm release.

Update V3.5 - Two-tier session prune + marathon save 3 May 2026

Distinguishes claude -p batch subprocesses (toolless, prune at 30 min) from interactive sessions (prune at 24 h). Marathon medals are saved at prune time so a 25h session caught during the next hook doesn't silently lose its marathon_24h.

Update V3.3 / V3.4 - 4-card PWA layout + collapsible categories 2 May 2026

Web view restructured into four distinct cards: PET (with derived Mood / Trait / Next evolution), CURRENT RUN (RUN + DEV lines), STATS, ACHIEVEMENTS. The achievements panel becomes seven collapsible categories with a virtual "Near completion" group that auto-appears for items past 70% target.

New V3.2 - 46 medal-tiered achievements 1 May 2026

13 medal families, each with bronze / silver / gold tiers (streak adds platinum at 30K XP), plus the 6-step hatch phase ladder. Backfill runs on every hook event - upgraders are credited retroactively for thresholds they already crossed.

New V2 - OpenTelemetry collector 30 Apr - 1 May 2026

OTLP/HTTP/JSON collector binds strictly to 127.0.0.1 on port 7879, ingests Claude Code's tokens / cost / lines / cache metrics, and unlocks 21 OTel-gated achievements (code lines, tokens, cache, frugal, big spender, PR, picky). Optional fan-out to Datadog / Honeycomb / Grafana Cloud via PETFORGE_OTEL_FORWARD.

New V1 - Initial design + implementation 30 Apr 2026

Hooks-based engine, 18 species, 6 evolution phases, lockfile state writes, ASCII rendering. Specs and implementation plan committed alongside code under docs/superpowers/.

03 / Tech Stack

Runtime, build, and platform

Single-file ESM, strict TypeScript, Vitest, Biome. No web framework, no bundler at runtime.

Runtime & language

Node.js≥20
ESM-only target. --experimental-vm-modules not needed; modern node: imports throughout.
TypeScript5.9.3
Strict mode, exhaustive switch checks, 1 explicit any across 10.8K LoC. Type errors fail prepublishOnly.
tsup8.5.1
esbuild-backed bundler. Outputs a single ESM dist/index.js at 382 KB - the entire CLI in one file.
Biome2.4.13
Linter + formatter in one Rust binary. Replaces ESLint + Prettier. Gates prepublishOnly.

Engine libraries

Ink7.0.1
React-for-CLIs from Vadim Demedes. Powers the watch animation, card snapshot, and cinematic level-up overlays. Renders into a Yoga flexbox layout.
React19.2.5
Required as Ink's renderer host. Concurrent features unused; this is a strict reconciler-only consumer.
Zod4.4.1
State schema validation on read with schemaVersion-tagged migrations. Guards against partial/corrupted state files.
proper-lockfile4.1.2
Cross-process advisory lock on ~/.petforge/.lock for the read-mutate-write cycle. Windows EPERM/EBUSY are retried.
chalk5.6.2
ANSI color rendering. Drives the rarity glow palette (common / uncommon / rare / epic / legendary) and shiny rainbow overlay.
figlet1.11.0
ASCII banner generation for cinematic level-ups, hatch milestones, and the card header.

Quality & testing

Vitest4.1.5
36 spec files, 461 test cases, 7K LoC of tests. Covers state migrations, hook events, achievement backfill, OTel ingest, quota probe, and per-platform service manager mocking.
ink-testing-library4.0.0
Headless renderer for Ink components. Lets the card / watch output be asserted against without a real TTY.

Infrastructure surface

Two local servers, both opt-in. Port 7878 binds to 127.0.0.1 by default (0.0.0.0 with --lan) and serves the HTML + SSE web view. Port 7879 always binds to 127.0.0.1 only and accepts OTLP/HTTP/JSON metric payloads. Optional shared-secret token authentication via --token=XXX. Optional OTel fan-out to a downstream collector (Datadog, Honeycomb, Grafana Cloud) via PETFORGE_OTEL_FORWARD. No vendor lock-in; no remote dependency for the core experience.

04 / Feature Inventory

What ships

14 CLI subcommands, 52 achievements, 18 species, 6 evolution phases, 1 PWA, 1 OTel collector.

Engine

Deterministic pet generation
sha256(username + hostname) seed picks rarity bucket first, then species + 5 derived stats. Same machine = same pet, always.
5 Claude Code hooks
UserPromptSubmit +5 / PostToolUse +1 / Stop +10 / SessionStart / SessionEnd +50. Zero polling.
Atomic locked state writes
proper-lockfile + write-to-temp + rename. Windows EPERM/EBUSY retry. Cross-process safe.
Zod-validated schema with migrations
schemaVersion-tagged state, migration chain on read. Forward-compatible additive fields.

Progression

6 evolution phases
Egg / Hatchling / Junior / Adult / Elder / Mythic - level 1 through 100, interpolated XP curve.
18 deterministic species
7 common / 4 uncommon / 4 rare / 2 epic / 1 legendary. Per-species ASCII generator.
Shiny variant (1%)
Independent rainbow overlay. Any species, any rarity.
52 achievements, 14 families
Bronze / silver / gold / platinum tiers. Idempotent backfill on every hook event.

Display surfaces

Terminal snapshot & live watch
8 FPS animation, 500ms state refresh, Ctrl+C / q to exit. Ink-rendered.
PWA web view (4 cards)
PET / CURRENT RUN / STATS / ACHIEVEMENTS + NEXT GOALS. Installable manifest + 512×512 icon.
Server-Sent Events stream
~50 ms phone update per hook event, auto-reconnect on disconnect. Read-only - never mutates state.
LAN access with shared-secret token
--lan + --host=IP + --token=XXX. Header or query param.

Integrations & ops

★Cross-platform service install
User-mode auto-start on Windows / macOS / Linux. No sudo. Idempotent. (V3.6)
★Quota tracker (5h + 7d)
Opt-in. JSONL-mtime gate prevents idle API calls. Mood flips stressed/panic at 80% / 95%. (V3.7)
★petforge history
Reads ~/.claude/projects/**/*.jsonl for lifetime spend, dedups by message.id. --sync-otel rewrites state counters. (V3.7)
OpenTelemetry collector
OTLP/HTTP/JSON on 127.0.0.1:7879. Fan-out to Datadog / Honeycomb / Grafana Cloud.
Buddy import (manual, consensual)
From file or stdin, max 32 KB. Auto-parses name / rarity / stars / stat lines. Strips parsed stats from rendered visual.
One-command orchestration
petforge up [--lan] [--quota] runs collect + serve (+ quota daemon), single Ctrl+C kills all.
Cinematic level-up & medal overlays
figlet-rendered banners on phase boundary + medal-tier unlocks. Web view and CLI.

Integration into AETHER OS
PetForge as an embeddable progression panel inside MindVision's local-first agentic runtime.
Position relative to the broader stack. PetForge is the first MindVision project to ship a tri-OS user-mode service installer and an OTel-driven achievement engine. Both pieces are designed as transplantable modules. The OS service code already sits behind a clean ServiceManager interface; lifting it into an internal library is a 2 - 3 day refactor.
